2.1 Review the Ongoing use of IT Tools and Change Approach as Needed in Making your Website

When making my website, the main tools I used were: Visual Studio Code, Trello, Pages and iCloud. Visual Studio Code is where most of the magic happens; so I will begin with this tool first. I had to ensure everything was coded correctly at the start, in order to begin creating my website. I enjoyed using Visual Studio Code, as it is a tool I am now becoming familiar with. There are a few interesting techniques I learnt to format my website in an organised way. For example, the use of 'strong' allowed me to make sentences other than my heading in a bold font.

Trello is a tool which needs constant updating, which is important as it allows me to update myself on what needs to be done. Trello is simple to use and helped me plan my day ahead more efficiently. I used Pages mainly for the context of my website, which I then added into Visual Studio. Lastly, iCloud was a great way to manage my project flexibly. For example, if I didn’t have my laptop on me. I could work on the context of my website using my phone as everything automatically updates on my devices.


2.2 Describe Whether the IT Tools Selected were Appropriate

Visual Studio Code was appropriate for my website as I was able to write my code speedily. Furthermore, being able to test my website was extremely helpful as I could correct anything I didn’t feel looked right. In addition to this, VSC picks up any errors, so if my code wasn’t appropriate, I was able to go over and check why - this was beneficial for my learning and practise.

Trello is appropriate for task management. The Kanban boards are shareable and flexible. Not only this, but each card allows you to add a great amount of information, which is good for detail. Lastly, it is easy to use, and a simple yet effective way of staying on top of tasks.
